# Alcance del Proyecto

## Objetivo General

Desarrollar una aplicación web para la gestión integral de proveedores, pedidos de producción e inventarios de materiales utilizados en la fabricación de casetones, permitiendo centralizar la información y mejorar el control operativo de la empresa.

---

# Módulo de Proveedores

Este módulo permitirá administrar la información de los proveedores de la empresa.

## Funcionalidades

* Registrar nuevos proveedores.
* Editar la información de proveedores existentes.
* Eliminar proveedores.
* Consultar la lista completa de proveedores.
* Buscar proveedores por nombre, empresa o identificación.
* Visualizar información detallada de cada proveedor.

## Datos del proveedor

* Nombre o razón social.
* NIT o identificación.
* Nombre del contacto.
* Número telefónico.
* Correo electrónico.
* Dirección.
* Estado (Activo/Inactivo).

---

# Módulo de Pedidos y Compras

Este módulo permitirá gestionar los pedidos relacionados con la producción de casetones y realizar seguimiento a su estado.

## Funcionalidades

* Registrar nuevos pedidos.
* Asociar pedidos a un proveedor.
* Actualizar el estado de producción de cada pedido.
* Consultar historial de pedidos.
* Buscar pedidos por fecha, proveedor o estado.
* Visualizar detalles completos del pedido.

## Estados de Producción

* Pendiente.
* En producción.
* En proceso de entrega.
* Entregado.
* Cancelado.

## Información del pedido

* Número de pedido.
* Fecha de creación.
* Proveedor asociado.
* Materiales solicitados.
* Cantidad.
* Estado actual.
* Observaciones.

---

# Módulo de Inventarios

Este módulo permitirá controlar los materiales utilizados para la fabricación de casetones y mantener actualizado el stock disponible.

## Funcionalidades

* Registrar materiales.
* Editar información de materiales.
* Eliminar materiales.
* Consultar inventario general.
* Actualizar cantidades disponibles.
* Visualizar alertas de stock bajo.
* Buscar materiales por nombre o categoría.

## Información del material

* Nombre del material.
* Categoría.
* Descripción.
* Unidad de medida.
* Cantidad disponible.
* Stock mínimo.
* Fecha de última actualización.

---

# Requerimientos Generales

* Sistema web accesible desde navegador.
* Interfaz intuitiva y fácil de usar.
* Gestión segura de la información.
* Base de datos para almacenamiento persistente.
* Validación de datos en formularios.
* Mensajes claros de éxito y error.
* Diseño adaptable a dispositivos móviles y computadores.

# Entregables

* Aplicación web funcional.
* Base de datos implementada.
* Documentación técnica básica.
* Manual de usuario.
* Código fuente del proyecto.
